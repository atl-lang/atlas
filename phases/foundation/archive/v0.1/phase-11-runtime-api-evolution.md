# Phase 11 - Runtime API Evolution Policy

## Objective
Define strict compatibility rules for the runtime API to avoid breaking embeddings.

## Inputs
- `docs/api/runtime-api.md`
- `docs/reference/versioning.md`
- `docs/runtime-api-evolution.md`

## Deliverables
- Backward compatibility policy for API changes.
- Deprecation rules and timeline.
- Versioning rules for API surface changes.

## Steps
- Define what counts as breaking vs non-breaking change.
- Document deprecation process and minimum support window.
- Add a checklist for API changes.

## Exit Criteria
- Policy is documented and referenced in README.
