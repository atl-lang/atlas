# Phase 03 - Module Scaffolding

## Objective
Prepare the codebase for a future module system without implementing it yet.

## Inputs
- `docs/modules.md`

## Deliverables
- Stub `modules` module with types and errors.
- Tests that confirm module system is disabled in v0.1 (error on `import`).

## Steps
- Add module-related enums and error codes.
- Add tests for `import` errors.

## Exit Criteria
- Clear errors if `import` is used in v0.1.
